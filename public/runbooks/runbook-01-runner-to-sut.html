Runbook 01 — Runner → SUT connectivity and naming

Prove a real runner-to-SUT network boundary and stable naming. Localhost must not be an acceptable target.

localhost negative proof DNS proof TCP transport proof remote HTTP positive proof automation proof

Objective + success criteria

Objective: Validate that automated tests execute from an isolated runner and reach the System Under Test (SUT) exclusively over a real network boundary using stable naming. “Localhost” must not be an acceptable target.

Success criteria:

  • Localhost negative proof: requests to http://localhost:${SUT_PORT} on the runner do not return the SUT application (connection refused/timeout is acceptable; 200 OK is a failure).
  • DNS proof: ${SUT_HOST} resolves to the intended SUT host.
  • TCP transport proof: TCP handshake to ${SUT_HOST}:${SUT_PORT} (or ${SUT_IP}:${SUT_PORT}) succeeds.
  • Remote positive proof: requests to ${BASE_URL} return an HTTP response from the SUT.
  • Automation proof: a Playwright test passes while targeting ${BASE_URL}.

✅ Sprint 1 — Completed (sanitized)

Sprint 1 is complete. Runner-to-SUT reachability is proven by name across a real network boundary, and localhost access is treated as a failure signal.

  • Runner resolves sut.testlab and reaches the SUT over TCP
  • Runner receives HTTP response from SUT via BASE_URL
  • Localhost is not a valid target (negative proof is enforced)
Private (operator): evidence blocks (actuals) 
# Runner DNS proof (macOS example)
dscacheutil -q host -a name sut.testlab

# Runner TCP + HTTP proof (hostname)
nc -vz sut.testlab 3001
curl -sS -D- http://sut.testlab:3001/ -o /dev/null | sed -n '1,12p'

# Localhost negative proof (runner)
curl -sv --max-time 2 http://localhost:3001/ | head -n 12

Localhost negative proof (runner)

Deterministic: this block FAILS if localhost returns HTTP, PASS if it refuses/timeouts. 

# PASS if curl fails (connection refused/timeout). FAIL if curl succeeds.
curl -fsS --max-time 2 "http://localhost:${SUT_PORT}/" >/dev/null \
  && { echo "FAIL: localhost returned HTTP (boundary bypass)"; exit 1; } \
  || echo "PASS: localhost did not respond (boundary enforced)"

Remote HTTP positive proof (runner)

curl -sS -I "${BASE_URL}/" | head -n 1
curl -sS -o /dev/null -D - --max-time 3 "${BASE_URL}/" | head -n 12

TCP transport validation (runner)

# hostname-first
nc -vz -w 2 "${SUT_HOST}" "${SUT_PORT}"

Red flag: localhost returns HTTP (boundary is being bypassed).

Store evidence as sanitized snippets or screenshots. Avoid publishing internal tokens, credentials, or network fingerprint details.

Failure modes + how to diagnose

Localhost succeeds unexpectedly

Symptom: localhost returns HTTP (negative proof fails).

Diagnosis: a local service is listening on the runner, or a proxy/forward is masking the boundary.

Remote hostname does not resolve

Symptom: DNS resolution fails or resolves to an unintended host.

Diagnosis (runner):

dscacheutil -q host -a name sut.testlab
nslookup sut.testlab || true

Action: correct the mapping (e.g., /etc/hosts on the runner) to the intended SUT.

TCP connection fails

Symptom: nc reports timeout or connection refused.

Diagnosis: service is down, port is not exposed, or traffic is blocked.

Action: verify service state on the SUT, confirm port exposure, and review firewall controls.

Service bound to loopback on SUT

Symptom: service responds locally on the SUT but is unreachable from the runner.

Diagnosis: service is listening on 127.0.0.1:${SUT_PORT} only.

Action: bind to 0.0.0.0 or the LAN interface and restart the service.

Why it matters (production relevance)

This validation eliminates false confidence caused by localhost coupling and shared execution environments. By enforcing a real network boundary early, subsequent automation results reflect deployment conditions: name resolution, routing, port exposure, and service availability.

Failures observed after this point represent genuine infrastructure or application issues rather than test-environment artifacts, improving diagnosability, change safety, and release confidence.
