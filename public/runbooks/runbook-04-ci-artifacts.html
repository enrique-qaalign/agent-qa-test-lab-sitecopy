Runbook 04 — CI artifacts and “debug without rerun”

Validate that CI publishes sufficient artifacts to diagnose failures without re-executing tests. Artifacts must be consistently available show the failure mode.

HTML report screenshots video trace deterministic storage

Objective + success criteria

Objective: Verify that CI pipelines publish artifacts required to diagnose test failures without rerunning the pipeline, and that artifact links remain available for review.

Success criteria:

  • CI job uploads an HTML report (or equivalent) for each run
  • Failed tests include screenshots and (where configured) video and trace
  • Artifact paths are deterministic and discoverable
  • Failure diagnosis is possible using artifacts + CI logs only

Exact commands

Local reproduction of artifact generation (Runner)

npx playwright test --project=Chromium --trace=on --screenshot=on

Open local report (Runner)

npx playwright show-report

Locate local artifacts (Runner)

ls -la test-results || true
ls -la playwright-report || true

CI validation (GitLab UI)

In the GitLab job page: open job artifacts and confirm presence of report + per-test artifacts.

Optional: enforce artifact existence in CI logs

test -d playwright-report || (echo "missing playwright-report" && exit 1)
test -d test-results || (echo "missing test-results" && exit 1)

Artifact generation flags may be configured in playwright.config.*. Use project defaults where applicable.

Expected outputs (logs, screenshots, artifacts)

playwright-report HTML report directory exists and contains index.html or equivalent entry point.
test-results Per-test artifact directories exist with failure evidence (screenshot/video/trace where enabled).
CI job artifacts Artifacts are attached to the job and downloadable via GitLab UI.
Diagnosis evidence Failure can be understood from report + artifacts + CI logs without re-execution.

Failure modes + how to diagnose

Artifacts missing from CI job

Symptom: CI job completes but no artifacts are attached.

Diagnosis: confirm artifact paths and job configuration.

# In repo
ls -la
ls -la playwright-report test-results || true

Action: update CI artifacts:paths to include the correct directories.

Artifacts generated locally but not uploaded in CI

Symptom: local run produces artifacts; CI run does not.

Diagnosis: compare Playwright config and CI environment variables.

cat playwright.config.* 2>/dev/null || true

Artifacts exist but are not useful

Symptom: report does not show actionable failure context.

Diagnosis: ensure traces/videos are enabled for failures and retention is configured.

# Example config keys (inspect actual config)
# use: { trace: 'on-first-retry' }  or trace: 'on'
# use: { video: 'on-first-retry' } or video: 'on'
# use: { screenshot: 'only-on-failure' }

Action: enable artifacts for failure diagnosis and keep retention consistent.

Artifacts expire before review

Symptom: artifacts are missing due to expiration.

Diagnosis: inspect CI artifact retention settings.

Action: increase artifact retention window for review workflows.

Why it matters (production relevance)

Artifact-first pipelines reduce cycle time and eliminate re-run dependence. When failures are diagnosable from artifacts alone, teams can triage accurately, reduce flaky reruns, and maintain trust in CI as a release signal.
