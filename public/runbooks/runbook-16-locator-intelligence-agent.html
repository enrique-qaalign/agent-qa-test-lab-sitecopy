Definition of done

  • ✔ One real page can be inventoried end-to-end.
  • ✔ Output includes role, accessible name, locator, and risk.
  • ✔ Ambiguous elements are flagged instead of forced into brittle selectors.
  • ✔ Generated recommendations are usable in the framework with minimal editing.

Locator priority model

Priority 1 getByRole(role, { name })
Priority 2 getByLabel()
Priority 3 getByPlaceholder()
Priority 4 getByText() only when stable and semantically anchored
Last resort CSS/XPath, explicitly flagged as higher risk
Risk model
  • LOW: unique role + unique accessible name
  • MEDIUM: stable semantics but repeated names or dynamic text dependence
  • HIGH: index-based, CSS chain, or missing accessibility semantics

Mechanics

  • Navigate to the target route in a known state.
  • Capture the accessibility tree.
  • Normalize nodes by role, name, and interactive value.
  • Generate ranked locator recommendations.
  • Attach ambiguity and accessibility risk flags.
Extraction 
const snapshot = await page.accessibility.snapshot({ interestingOnly: false });

{

Do
  • Prefer accessibility-driven selectors.
  • Expose ambiguity explicitly.
  • Use the output as a locator inventory and as a product accessibility signal.
Don’t
  • Do not default to CSS because it is faster to type.
  • Do not hide missing labels or duplicate names.
  • Do not treat DOM depth as meaning.

Expected outputs

  • ✔ Page inventory of interactive controls.
  • ✔ Ranked locator recommendations tied to accessibility semantics.
  • ✔ Risk flags for duplicate names and missing labels.
  • ✔ Reusable machine-readable locator catalog.

Failure modes

  • Accessibility tree incomplete: action: ensure the page is in the right state before snapshot.
  • Multiple elements share role/name: action: annotate ambiguity and suggest stronger semantic anchor.
  • Team falls back to CSS anyway: action: preserve risk labels and policy note.
  • Dynamic content makes text unstable: action: parameterize test data or favor role+label on controls.

Operational value

  • Faster authoring of stable Playwright locators.
  • Consistent accessibility-first locator policy.
  • Reusable page inventory for framework growth.
  • Better UI diagnosability when locator failures occur.
