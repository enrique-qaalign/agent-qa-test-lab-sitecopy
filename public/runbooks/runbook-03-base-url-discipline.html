Runbook 03 — BASE_URL discipline and proof

Validate that environment configuration controls navigation targets and that test code does not hardcode domains or rely on implicit defaults.

env-driven config fail-fast validation no hardcoded hosts deterministic targeting

Objective + success criteria

Objective: Verify that the target environment is provided by BASE_URL (or the project’s configured equivalent) and that missing or incorrect values produce fast, diagnosable failures.

Success criteria:

  • When BASE_URL is unset, validation fails immediately with a clear error
  • When BASE_URL is incorrect, navigation fails with an external symptom (DNS/timeout/connection error)
  • When BASE_URL is correct, tests target the intended SUT host
  • Test code contains no hardcoded hostnames for environment selection

Exact commands

Inspect configured target (Runner)

node -e 'console.log("BASE_URL=", process.env.BASE_URL || process.env.BASE_URL_TODO || "")'

Unset BASE_URL and validate fail-fast behavior (Runner)

unset BASE_URL
node -e 'console.log("BASE_URL=", process.env.BASE_URL || "")'

Run a single test with BASE_URL unset (Runner)

npx playwright test -g "TODO\.NEW\.001" --project=Chromium --headed

Set an incorrect BASE_URL and run (Runner)

export BASE_URL="http://sut.testlab:3999"
npx playwright test -g "TODO\.NEW\.001" --project=Chromium --headed

Set correct BASE_URL and run (Runner)

export BASE_URL="http://sut.testlab:3001"
npx playwright test -g "TODO\.NEW\.001" --project=Chromium --headed

Optional: verify HTTP reachability matches BASE_URL (Runner)

curl -sS -o /dev/null -D - --max-time 3 "${BASE_URL}/" | head -n 12

If the project uses a different variable (e.g., BASE_URL_TODO), substitute accordingly.

Expected outputs (logs, screenshots, artifacts)

BASE_URL unset Test run fails early with a clear configuration error (no navigation attempts required).
BASE_URL incorrect Failure occurs at DNS, connection, or timeout layer and points to an unreachable target.
BASE_URL correct Navigation targets the intended SUT host and proceeds to application-level assertions.
Evidence to capture Console echo of BASE_URL, relevant error output, and Playwright artifacts if enabled.

Sanitize hostnames and LAN IPs if publishing outputs.

Failure modes + how to diagnose

BASE_URL not enforced (silent fallback)

Symptom: tests run using an implicit default target when BASE_URL is missing.

Diagnosis: inspect config and test logs for implicit navigation targets.

node -e 'console.log("BASE_URL=", process.env.BASE_URL || process.env.BASE_URL_TODO || "")'
grep -R "http://" -n pages tests playwright.config.* || true
grep -R "https://" -n pages tests playwright.config.* || true

Incorrect variable name used

Symptom: BASE_URL is set but framework reads a different variable.

Diagnosis: search for environment reads.

grep -R "process\.env\." -n pages tests playwright.config.*

BASE_URL correct but connectivity fails

Symptom: correct target value; network errors occur.

Diagnosis: validate reachability outside of Playwright.

curl -sv --max-time 3 "${BASE_URL}/" || true
nslookup sut.testlab || true
nc -vz -G 2 192.168.0.31 3001 || true

Why it matters (production relevance)

Environment-driven targeting prevents test code from becoming configuration drift. It supports deterministic execution across developer machines, CI runners, and staged environments without code changes, reducing misconfiguration risk and improving diagnosability.
