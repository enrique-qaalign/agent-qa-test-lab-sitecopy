# QA ALIGN Certified Architect Checklist

## Purpose

The Architect level proves that the engineer can assess systems, route sprint paths, estimate remediation, govern trust, and approve what may enter trusted release signal.

This is the leadership and delivery-governance level.

---

## Competencies

### Assessment and Routing
- Can assess a framework using workbook/model inputs
- Can validate trust-level assignment
- Can explain dominant risk families
- Can justify start sprint and sequence
- Can explain when Sprint 3 should precede Sprint 4
- Can explain no_ai vs some_ai vs ai_forward decisions

### Commercial and Remediation Judgment
- Understands remediation pricing model
- Can estimate hours by category
- Can explain severity, scope, and framework multipliers
- Can translate technical findings into proposal ranges
- Can distinguish per-occurrence cleanup from systemic package work

### Governance
- Can define what may enter release signal
- Can define what remains draft-only
- Can set approval thresholds for generated assets
- Can explain how release trust is protected
- Can define safe AI boundaries for a team

### Leadership
- Can review Builder decisions
- Can explain QA ALIGN clearly to clients and hires
- Can identify when a system is not ready for AI-forward generation
- Can certify Operators and Builders
- Can coach others on trust-protection decisions

---

## Architect Practical Exam

### Practical 1 — Full System Assessment
Given:
- workbook-derived input
- framework assessment
- sprint routing
- advisory report

Task:
- validate or override trust level
- validate or override sprint route
- explain the path recommendation
- identify where automation is correct and where human override is needed

Pass criteria:
- reasoning is coherent
- candidate protects the system from unsafe conclusions
- candidate can justify overrides

---

### Practical 2 — Remediation Estimate
Given:
- anti-pattern matrix with pricing model
- hourly rate = 215

Task:
- estimate remediation hours by category
- estimate target and high ranges
- identify which categories should be packaged as systemic work
- explain the commercial meaning of the estimate

Pass criteria:
- estimate is plausible
- cost drivers are understood
- uncertainty is communicated clearly

---

### Practical 3 — Governance Decision
Given:
- generated page object
- generated specs
- confidence scores
- advisory report

Task:
Classify each artifact as:
- candidate only
- ready for review
- ready for stabilization
- eligible for promotion into smoke
- not eligible for release-signal influence

Pass criteria:
- trust is protected correctly
- candidate does not promote weak artifacts too early
- rationale is evidence-based

---

### Practical 4 — Coaching Simulation
Task:
Explain to:
1. a junior engineer why a negative generated test remains review-required
2. a client why low trust does not justify AI-forward automation yet
3. leadership why Sprint 3 may need to come before Sprint 4

Pass criteria:
- explanations are technically precise
- explanations are commercially credible
- explanations are aligned with QA ALIGN principles

---

## Scoring Rubric

### Assessment quality
30 percent

### Governance judgment
30 percent

### Remediation estimation
20 percent

### Coaching and communication
20 percent

---

## Pass Rule

- Minimum 90 percent overall
- Must pass Practical 1 and Practical 3
- Must not make unsafe promotion decisions
- Must show Architect-level judgment

---

## Certification Decision

### Pass
Candidate may govern QA ALIGN delivery, approve promotion boundaries, and certify lower levels.

### Conditional Pass
Candidate may perform Architect work under review until specified gaps are corrected.

### Fail
Candidate requires further development before reassessment.
