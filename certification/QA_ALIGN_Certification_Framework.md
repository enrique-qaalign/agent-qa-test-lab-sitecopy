# QA ALIGN Certification Framework

## Purpose

QA ALIGN certification proves that an engineer can operate inside the QA ALIGN Automation System with discipline, consistency, and sound judgment.

This certification is not a generic tool badge.
It validates the ability to work within a deterministic QA system that emphasizes:

- environment discipline
- artifact-first debugging
- anti-pattern awareness
- sprint-based remediation
- controlled AI usage
- release-signal protection

---

## Certification Levels

### 1. QA ALIGN Automation System Certified — Operator
Can run the QA ALIGN pipelines, interpret outputs, and handle generated artifacts safely.

### 2. QA ALIGN Automation System Certified — Builder
Can build, review, improve, and correct automation inside the QA ALIGN system.

### 3. QA ALIGN Automation System Certified — Architect
Can assess systems, route sprint paths, estimate remediation, govern trust, and certify others.

---

## Core Principles

All certified engineers must understand and respect these rules:

1. Generated artifacts are not automatically trusted release signal.
2. Deterministic evidence is required before promotion.
3. Low-confidence outputs require review.
4. Anti-patterns must be identified and routed intentionally.
5. AI may assist generation, but trust must be earned through validation and execution.
6. Release decisions must remain evidence-driven.

---

## Certification Structure

Each level includes:

- checklist
- practical exam
- evaluator review
- pass/fail decision
- certification record

---

## Recertification

### Operator
Every 12 months

### Builder
Every 12 months

### Architect
Every 6 to 12 months depending on framework evolution

---

## Permission Model

### Operator permissions
- run pipelines
- review reports
- inspect generated artifacts
- identify issues for escalation

### Builder permissions
- modify generated artifacts
- improve locators, methods, composites, and tests
- correct anti-patterns
- stabilize draft assets

### Architect permissions
- approve promotion into trusted suites
- define AI boundaries
- set remediation plans
- approve sprint routing
- certify Operators and Builders

---

## Required Evidence

A certification is valid only when the candidate has completed the practical work and been evaluated against QA ALIGN standards.

A quiz alone is not sufficient.

## Pass Thresholds

### Operator
- 80 percent overall
- must pass core pipeline operation
- must not violate process discipline

### Builder
- 85 percent overall
- must pass artifact review and remediation practicals
- must not approve unsafe output

### Architect
- 90 percent overall
- must pass governance and routing practicals
- must not make unsafe promotion decisions

---

## Operating Statement

A QA ALIGN certified engineer does not merely know a test tool.
A QA ALIGN certified engineer knows how to operate within a deterministic automation system that protects release confidence.
